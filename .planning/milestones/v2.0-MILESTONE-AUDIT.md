# v2.0 MCP Apps — Milestone Audit Report

**Milestone:** v2.0 MCP Apps
**Audited:** 2026-03-24
**Status:** gaps_found
**Phases:** 6/6 complete (Phases 8-13)

## Executive Summary

25 of 27 v2.0 requirements are fully satisfied with verified evidence. Two requirements have integration-level gaps:

1. **TOOL-03** (waf_approve_gate): The tool exists and unit tests pass, but the interactive pipeline flow is not functional end-to-end. The pipeline runner has no interactive mode implementation, and the gate response file paths don't connect.

2. **QUAL-02** (E2E Playwright gate): Fully implemented with graceful degradation, but `playwright` is not declared as a dependency. The gate is effectively unavailable on fresh installations.

Three of four E2E user flows work correctly. The interactive gate approval flow is broken.

## Integration Check Results

### Connected (25/27 requirements fully wired)

All MCP tools import and use their backing modules correctly. The deploy_target parameter flows end-to-end from `waf_generate_app` → `_pipeline_bridge` → `run_pipeline` → `PhaseContext.extra` → `get_provider()`. All 7 MCP tools are registered and callable. The dev server lifecycle, environment checker, and credential management are fully connected.

### Broken Flows (2 issues)

**BREAK-01: Interactive mode silently dropped (HIGH)**
- `_pipeline_bridge.py` converts `mode="interactive"` to `dry_run=False` and discards the interactive signal
- `contract_pipeline_runner.py:run_pipeline()` has no `mode` or `interactive` parameter
- The pipeline always runs in auto mode regardless of the user's mode selection
- Affects: TOOL-03

**BREAK-02: Gate response file path mismatch (HIGH)**
- `waf_approve_gate` writes to `output/.gate-responses/{run_id}.json`
- `mcp_approval_gate.approve_gate` polls `/tmp/factory_approve_response_{pid}-{uuid}`
- No code bridges these two paths
- Affects: TOOL-03

**GAP-01: Playwright undeclared dependency (MEDIUM)**
- `playwright` not in `pyproject.toml` dependencies
- `e2e_gate.py` has graceful import guard (returns BLOCKED, not crash)
- Gate is effectively disabled on fresh installations
- Affects: QUAL-02
