<objective>
Wire interactive_mode from the MCP bridge through the pipeline runner to the gate, and validate the full end-to-end flow.

Purpose: Close BREAK-01 (mode='interactive' silently dropped). The bridge currently tracks `mode` in ProgressStore but never forwards it to `run_pipeline()`. This plan adds `interactive_mode` as a keyword argument through the call chain: bridge -> run_pipeline -> _run_gate_checks -> run_mcp_approval_gate(interactive=True). It also emits a `gate_waiting` progress event so waf_get_status shows the pipeline is paused. Finally, an integration test proves the full approve/reject flow works end-to-end.

Output: Updated _pipeline_bridge.py and contract_pipeline_runner.py with interactive_mode wiring, comprehensive unit tests, and a threading-based integration test.
</objective>
