    Add `interactive` and `run_id` keyword-only parameters to `run_mcp_approval_gate`:

        phase_id: str,

    When `interactive=True`:
    - If `run_id` is empty, return `GateResult(passed=False, status="BLOCKED", issues=["Interactive mode requires run_id..."])`
    - Otherwise, delegate to `_poll_mcp_gate_file(phase_id, run_id)`

    When `interactive=False`: keep existing `asyncio.run(approve_gate(...))` behavior unchanged (backward compatibility for `factory.py` CLI).

    Implement `_poll_mcp_gate_file(phase_id, run_id, *, poll_interval=2.0, timeout_seconds=0) -> GateResult`:
    - Import `GATE_RESPONSES_DIR` from `config.settings`
    - Compute `gate_file = GATE_RESPONSES_DIR / f"{run_id}.json"`
    - Poll loop: `while True: check file exists, try read+parse JSON, handle approve/reject, sleep(poll_interval)`
    - Use `try/except (FileNotFoundError, json.JSONDecodeError, OSError)` around read — treat errors as "not yet available"
    - On successful read, call `gate_file.unlink(missing_ok=True)` to consume the file
    - If `timeout_seconds > 0` and elapsed >= timeout, return GateResult(passed=False, status="BLOCKED", issues=["Timeout..."])
    - Timeout default of 0 means no timeout (poll indefinitely, matching factory_mcp_server.approve_gate behavior)
