def run_static_analysis_gate(project_dir: str, phase_id: str = "2b") -> GateResult:

Implements two checks per CONTEXT.md locked decisions (regex-based, not AST):

**Check 1 (GATE-05): `no_use_client_in_layout`**
- Scan ONLY `src/app/layout.tsx` and `src/app/page.tsx` (exactly two files — NOT error.tsx, NOT nested routes, NOT components)
- Regex: `r"""['"]use client['"]"""`
- On match: add issue with file path (relative to project_dir) and line number

**Check 2 (GATE-06): `no_next_public_secrets`**
- Scan all files under `src/` with extensions: `.ts`, `.tsx`, `.js`, `.jsx`, `.env`, `.env.local`
- Also scan root `.env` and `.env.local` files
- Regex: `r"NEXT_PUBLIC_(?:.*KEY|.*SECRET|.*TOKEN)"` — matches KEY, SECRET, or TOKEN anywhere after NEXT_PUBLIC_
- Skip `node_modules/` directory entirely
- On match: add issue with file path and line number

Combine issues from both checks. Return GateResult(passed=len(issues)==0) with all issues listed.

Tests create real temporary files with specific content (no mocking needed — the gate reads files). Use `tmp_path` fixture to create test file trees.
