<feature>
  <name>Dev Server Lifecycle Management</name>
  <files>web_app_factory/_dev_server.py, tests/test_dev_server.py</files>
  <behavior>
    LDEV-01 — Process spawning:
    - start_dev_server("run-123") calls subprocess.Popen with ["npm", "run", "dev", "--", "--port", "0"]
    - Popen uses start_new_session=True, stdout=PIPE, stderr=STDOUT, text=True
    - cwd is set to the resolved project directory from run_id

    LDEV-02 — Readiness detection:
    - Background thread reads stdout line-by-line
    - Regex `r"-\s+Local:\s+(http://localhost:(\d+))"` captures URL and port from "- Local: http://localhost:XXXX"
    - Regex `r"(?:✓\s+)?Ready in \d+"` confirms readiness
    - threading.Event is set when BOTH URL is captured AND Ready pattern matches
    - Returns formatted markdown: "✓ Dev server started\n\n**URL:** {url}\n**PID:** {pid}\n**Run ID:** `{run_id}`"
    - If 30s timeout expires before ready: returns error message, process is terminated
    - If process exits before ready: returns error with exit code

    LDEV-03 — Duplicate prevention and stale detection:
    - If run_id already in registry AND os.kill(pid, 0) succeeds: return existing URL without new Popen
    - If run_id in registry BUT os.kill(pid, 0) raises ProcessLookupError: remove stale entry, start new server
    - DevServerInfo is frozen dataclass: run_id, pid, port, url, started_at, project_dir
    - Registry is thread-safe via threading.Lock

    LDEV-04 — Cleanup:
    - stop_dev_server(run_id) sends SIGTERM to process group via os.killpg(pgid, SIGTERM)
    - After 5s timeout, escalates to SIGKILL if process still alive
    - atexit handler iterates all tracked servers, calling _terminate_server on each
    - Signal handlers for SIGTERM/SIGINT also trigger cleanup (wrapped in try/except ValueError for non-main-thread)
    - _PROC_MAP: dict[str, subprocess.Popen] separate from frozen DevServerInfo registry
    - Separate from _REGISTRY because Popen is mutable and cannot be in frozen dataclass

    Project dir resolution:
    - Scan output/*/docs/pipeline/runs/{run_id}/state.json
    - Read "project_dir" key from JSON
    - Return None if run_id not found (error message returned to caller)
  </behavior>
  <implementation>
    After tests pass, ensure:
    - Module-level singleton: _REGISTRY = DevServerRegistry(), get_registry() accessor
    - Module-level _PROC_MAP: dict[str, subprocess.Popen] for mutable Popen references
    - atexit.register(_cleanup_all_servers) at module level
    - signal.signal(SIGTERM/SIGINT, _signal_handler) wrapped in main-thread check
    - _READINESS_TIMEOUT = 30.0 as module constant
    - All subprocess calls use list args (no shell=True) — enforced by test_subprocess_audit.py
    - text=True with errors='replace' on Popen for encoding safety
  </implementation>
</feature>
