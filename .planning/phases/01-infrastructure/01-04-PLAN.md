       `load_contract(contract_path: str) -> dict` — loads YAML, validates against JSON schema (CONT-03)

       `run_pipeline(contract: dict, project_dir: str, idea: str, resume_from: str | None = None, dry_run: bool = False, skip_gates: bool = False) -> dict`:
       - Initialize run via pipeline_state.init_run()
       - If resume_from, skip completed phases
       - For each phase in PHASE_ORDER:
         - Look up executor from registry
         - Call pipeline_state.phase_start()
         - Execute phase via executor.execute()
         - If phase fails, call pipeline_state.mark_failed() and stop
         - If phase succeeds, generate quality self-assessment via generate_quality_self_assessment() (CONT-04)
         - Run gate checks
         - If gate fails and not skip_gates, stop pipeline
         - Call pipeline_state.phase_complete()
       - Return summary dict

       Keep the file under 400 lines. If the ios-app-factory runner is too complex, simplify: no rejection-fix mode, no iOS substep logging, no Gemini backend support. Focus on the core loop: load contract -> iterate phases -> execute -> gate -> persist.
