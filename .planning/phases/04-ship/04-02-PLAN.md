    - Passes when all 4 required headers present

    - Filters violations to impact == "critical" only (per GATE-07)
    - Zero critical violations -> pass
    - One or more critical violations -> fail with violation descriptions as issues
    - Stores full violation count in extra={"total_violations": N, "critical_violations": N}

    Link integrity gate:
    - run_link_integrity_gate(url, phase_id="3") returns GateResult(gate_type="link_integrity")
    - BFS crawl of deployed app starting from url
    - Extracts internal links (same hostname) from HTML
    - Checks each internal link returns 200 or 301/302/308
    - 404 responses -> issues
    - Max crawl depth of 3 levels (prevent infinite crawl)
    - Max 50 URLs to check (prevent runaway on large apps)
    - Uses httpx for all requests
    - Stores checked URLs count in extra={"urls_checked": N}
  </behavior>

       - Import from axe_playwright_python.sync_playwright: Axe
       - Launch chromium(headless=True), navigate to url with wait_until="networkidle", timeout=30000
       - Run axe.run(page), filter violations by impact == "critical"
       - Zero critical -> GateResult(passed=True, ...)
       - Critical violations found -> GateResult(passed=False, issues=[...])
       - Catch PlaywrightError and Exception -> GateResult(passed=False, issues=[error_msg])
       - Per RESEARCH.md Pitfall 4: use sync_playwright (not async) to avoid event loop conflict

       - Timeout per request: 10 seconds
