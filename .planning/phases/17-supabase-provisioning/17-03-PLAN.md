1. Create `web_app_factory/_supabase_provisioner.py` (~200 lines):
   - `SupabaseProvisioner` class with `__init__(self, access_token: str, org_id: str)`
   - Credential values NEVER logged (per security contract). Only operation names and status.
   - `_headers()` returns auth headers
   - `async create_project(self, name: str) -> dict`:
     - Generate db_pass with `secrets.token_urlsafe(32)`
     - POST to `https://api.supabase.com/v1/projects` with name, organization_slug, db_pass, region selection (americas smartGroup)
     - Return response JSON with `_db_pass` added (ephemeral, never serialized)
   - `async poll_until_healthy(self, ref: str, *, timeout_s: int = 300, interval_s: int = 5) -> None`:
     - GET `/v1/projects/{ref}/health` in a loop
     - All services must have `status == "ACTIVE_HEALTHY"`
     - Empty list = not yet healthy (Pitfall 5 from research)
     - Raise `TimeoutError` if deadline exceeded
   - `async get_api_keys(self, ref: str) -> dict[str, str]`:
     - GET `/v1/projects/{ref}/api-keys`
     - Return `{"anon_key": ..., "service_role_key": ...}`
   - `async inject_vercel_env(self, vercel_token: str, vercel_project_id: str, supabase_url: str, anon_key: str, service_role_key: str) -> None`:
     - POST to `https://api.vercel.com/v10/projects/{id}/env?upsert=true`
     - Set NEXT_PUBLIC_SUPABASE_URL (plain, all targets)
     - Set NEXT_PUBLIC_SUPABASE_ANON_KEY (plain, all targets)
     - Set SUPABASE_SERVICE_ROLE_KEY (sensitive, production+preview only — NO NEXT_PUBLIC_ prefix)
   - Uses `httpx` (already in deps) for all HTTP calls

2. Create `web_app_factory/_supabase_migration.py` (~120 lines):
   - `generate_migration_sql(entities: list[dict]) -> str`:
     - `entities` is a list of dicts with `name` (table name) and `columns` (list of column dicts)
     - Every table gets `user_id uuid REFERENCES auth.users NOT NULL`
     - Every table gets `ENABLE ROW LEVEL SECURITY`
     - Every table gets 4 CRUD policies using `(SELECT auth.uid()) = user_id` pattern (with SELECT subquery wrapper per research anti-pattern note)
     - Every table gets `CREATE INDEX ON public.{table} (user_id)` (Pitfall 4)
     - Policies use `TO authenticated` role (not `TO public`)
   - Helper: `_generate_table_sql(entity: dict) -> str` for per-table SQL generation
   - Helper: `_generate_rls_policies(table_name: str) -> str` for the 4 CRUD policies
