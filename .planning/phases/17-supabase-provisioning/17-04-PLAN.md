<objective>
Wire SupabaseProvisioner and dual client templates into the waf_generate_app pipeline via phase_3_executor.py.

Purpose: Plans 01-03 build the Supabase modules in isolation. This plan connects them to the live pipeline so that `waf_generate_app` with a Supabase-enabled spec automatically provisions a Supabase project, renders the dual client TypeScript files into the generated app output, and runs the supabase_gate to verify everything. Without this wiring, the modules exist but are never called.

Output: Extended phase_3_executor.py with Supabase sub-steps, a template renderer that emits .ts files into the generated app, and integration tests proving the pipeline wiring works.
</objective>
