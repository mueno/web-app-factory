  <action>
    RED phase: Create `tests/test_env_checker.py` with all test cases listed above. Tests import from `web_app_factory._env_checker` and mock `shutil.which`, `subprocess.run`, `sys.platform`, and `os.environ`. Use `unittest.mock.patch` for mocking. All tests must fail initially (module does not exist).

    GREEN phase: Create `web_app_factory/_env_checker.py` implementing:

    1. **ToolStatus structure**: Each check returns a dict with keys: tool, status, version_found, version_required, install_command, note. Status is one of: 'present', 'outdated', 'missing', 'present_unauth'.

    2. **check_env(deploy_target: str) -> list[dict]**: Calls `_check_nodejs`, `_check_npm`, `_check_python_version` from `pipeline_runtime.startup_preflight` (intentional private import, documented). Conditionally calls `_check_vercel_cli` for deploy_target='vercel' and a new `_check_gcloud` wrapper for deploy_target='gcp'. Converts each raw result dict to a ToolStatus dict via `_build_tool_status()`. For vercel with deploy_target='vercel', also checks auth status via VERCEL_TOKEN env var / keychain lookup.

    3. **_INSTALL_COMMANDS dict**: Static lookup table keyed by (tool_name, platform_key) where platform_key is 'darwin' or 'linux'. Contains human-readable install commands for node, npm, vercel, gcloud. npm entry says "npm comes with Node.js -- install Node.js first".

    4. **_INSTALL_ARGS dict**: Subprocess-safe list-form args for tools that support single-command install (vercel on both platforms: ['npm', 'install', '-g', 'vercel']; node on darwin: ['brew', 'install', 'node']). Missing entries (e.g., node on linux) mean "cannot auto-install".

    5. **_INSTALLABLE_TOOLS frozenset**: Allowlist of valid tool names for install_tool(): {'node', 'npm', 'vercel', 'gcloud'}. Any name not in this set is rejected with an error message BEFORE any subprocess call (security: prevents injection via tool_to_install parameter per Pitfall 3 in RESEARCH.md).

    6. **install_tool(tool_to_install: str) -> str**: Validates against allowlist, looks up _INSTALL_ARGS for platform, runs `subprocess.run(args, capture_output=True, text=True, timeout=120, check=False)` with list-form args (NO shell=True). Returns success/failure message. If no _INSTALL_ARGS entry exists for the tool+platform combo, returns human-readable manual install instructions from _INSTALL_COMMANDS.

    7. **format_env_report(statuses: list[dict], install_result: str | None = None) -> str**: Renders a markdown table with columns: Tool, Status, Version Found, Required, Install Command. Appends "Environment is ready" if all present, or install instructions if any missing/outdated. Appends notes (e.g., Vercel scope warning) below the table. Optionally appends install_result section.

    8. **_check_gcloud wrapper**: For gcloud detection, use `shutil.which("gcloud")` for presence, `subprocess.run(["gcloud", "--version"])` for version extraction. For auth: import and call `_check_gcloud_auth` from `tools.deploy_providers.gcp_provider`. Wrap all in try/except for robustness.

    9. **Python status simplification** (per Pitfall 6 in RESEARCH.md): Python is always 'present' when the MCP server is running. Report `sys.version` as version_found.

    10. **Vercel auth detection**: Check `os.environ.get("VERCEL_TOKEN")` first, then `get_credential("vercel_token")` from `_keychain.py`. If token found: status='present' with note="Warning: Vercel tokens cannot be project-scoped -- all projects in your org are accessible." If no token: status='present_unauth' with note="VERCEL_TOKEN not set. Run: vercel login OR set VERCEL_TOKEN env var."

    REFACTOR phase: Ensure all functions have docstrings, type hints are complete, and no `shell=True` exists anywhere in the file. Run `tests/test_subprocess_audit.py` to confirm the new file passes the static shell=True scan.
